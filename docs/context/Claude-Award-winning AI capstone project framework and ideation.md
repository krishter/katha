# AI PM Capstone Strategy: Framework & Ideation Guide

## Phase 1: What Evaluators Actually Value (Evidence-Based Framework)

Based on analysis of award-winning capstone projects and PM evaluation criteria, here's what separates excellent from mediocre execution in each phase:

### **Discovery Phase Excellence**
*What evaluators look for:*
- **Quantified problem**: Not "users struggle with X" but "72% of users abandon after 3 failed attempts, costing $2.4M annually"
- **Market validation**: Even if synthetic, show you interviewed 5-8 stakeholders or analyzed existing usage data
- **Clear AI appropriateness**: Why AI vs. rules-based? Show you considered alternatives
- **Risk identification early**: Flag privacy/bias/hallucination risks in Discovery, not as afterthoughts

*Strong artifact signals:*
- Value Map with actual numbers (TAM/SAM/SOM even if estimated)
- Competitive analysis showing white space, not just "X exists but isn't AI-powered"
- User research summary with direct quotes demonstrating pain depth

### **Design Phase Excellence**
*What evaluators look for:*
- **AI interaction patterns**: Beyond input→output, show you understand conversational design, progressive disclosure, confidence indicators
- **Failure state design**: How does your product handle hallucinations, ambiguous queries, edge cases?
- **Multi-stakeholder considerations**: B2B products especially - who configures, who uses, who pays?

*Strong artifact signals:*
- Wireframes showing 5+ distinct interaction states (not just happy path)
- Explicit annotation of where AI adds value vs. deterministic logic
- Error handling flows documented

### **Develop Phase Excellence**
*What evaluators look for:*
- **Prompt engineering depth**: Show iteration - v1 vs v2 vs v3 of your master prompt with specific improvements
- **Evaluation rigor**: Not "it works well" but "scored 8.2/10 on factual accuracy across 20 test cases"
- **Constraint handling**: How you dealt with context limits, latency, cost per request

*Strong artifact signals:*
- Master Prompt with inline comments explaining design choices
- Evaluation set with 15-20 diverse test cases covering edge cases
- Performance benchmark table (accuracy, latency, cost metrics)

### **Deploy Phase Excellence**
*What evaluators look for:*
- **Launch realism**: Phased rollout, not "launch to everyone on day 1"
- **Metric sophistication**: Leading indicators (engagement) + lagging indicators (business outcome)
- **Risk mitigation specifics**: "How will you detect prompt injection?" not "we'll monitor for security issues"

*Strong artifact signals:*
- Launch checklist with go/no-go criteria
- Success metrics tied back to Discovery phase value proposition
- 30/60/90 day roadmap showing iteration plan
